**Оформление PR**

Название (title) пулл-реквеста (PR) должно иметь формат:

`	`**<Фамилия>\_<Имя>\_<Вид работы><Номер работы>**

`   `Пример именования ветки для студента Иванова Ивана, который хочет выполнить первую лабораторную работу:		Ivanov\_Ivan\_lb1

Детали оформления см. в соотв. файле и по ссылке <https://se.moevm.info/doku.php/courses:github_rules> 

Комментарии коммитов должны быть осмысленными, например, "Ivanov LB1 done" 

Каждая новая лабораторная должна находиться в своей ветке, которая обязательно должна быть создана из ветки main

Каждая лабораторная работа должна находиться в своей папке, которая должна называться также, как ветка  

В репозитории должен храниться **только исходный код и файлы журналирования выполнения и самой лабораторной и отдельных программ (лог файлы).**  
Внутри них можно дополнять свои комментарии

Не добавляйте никакие другие файлы в Ваш PR (пуллреквест)

Файлы с исходным кодом должны находиться в папке **src** внутри вашей папки с лабораторной

Все добавляемые/изменяемые/удаляемые файлы должны относиться только **к рабочей папке**, вне нее ничего менять нельзя (в том числе удалять чужие рабочие папки, добавлять свои документы в корень репозитория)
Нельзя добавлять файлы с запрещенными расширениями. К запрещенным расширениям относятся
.pyc, .o, .exe, .out

В корневой папке репозитория должна быть папка с названием, совпадающим с названием PR

**Оформление кода и логов**

**Исходный код** – традиционно + **собственные комментарии,** (! уметь **модифицировать** код на занятии)

**Логи** – файлы журналирования для каждой лаб.работы:
Логи выполнения лаб.работы – **один общий** и можно выделить логи **по отдельным** объемным **пунктам** программы лаб.работы,

**Общ.лог** обязательно начать с информации о **профиле системы**: инф.о ядре, версии ОС, виртуализации, аппаратуре (кол-во процессорных ядер) и т.п., для этого использовать утилиты и команды ОС, отклик включить в лог, указать дату на момент исполнения 
Любая информацияв д.б. получена **средствами ОС**

Поясняющие **Комментарии** по ходу логов **обязательны**
**Результаты в логе** д.б. **пронумерованы** в соответствии с нумерацией в **программе работы,** 
как и **именование логов** должно **соответствовать выполняемому пункту.**
**Недопустимо** в логе **размещение справочной информации** и **фрагментов документации,** можно разместить **ссылку.**
